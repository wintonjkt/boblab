What Are Custom Modes?

Custom modes are predefined configurations that shape Bob's behavior, context, and capabilities for specific scenarios.

Mode Components

  • System Prompts: Define Bob's role and behavior
  • Context Templates: Pre-configured context for specific tasks
  • Tool Preferences: Optimize tool selection for the mode
  • Response Formats: Standardize output structure
  • Guardrails: Apply specific constraints and guidelines

Understanding Built-in Modes

Bob IDE comes with five powerful built-in modes, each optimized for specific workflows. Understanding when and how to use each mode is key to maximizing your productivity.

Why Use Different Modes?

Task Specialization

Get precisely the type of assistance needed for your current task

Safety Controls

Prevent unintended file modifications with mode-specific restrictions

Focused Interactions

Responses optimized for your current activity

Workflow Optimization

Seamless transitions between planning, implementing, debugging, and learning

Switching Between Modes

Bob provides four convenient ways to switch modes:

  • Dropdown Menu: Click the mode selector to the left of the chat input
  • Slash Commands: Type /plan, /ask, /code, /advanced, or /orchestrator
  • Keyboard Shortcut: Press ⌘+. (macOS) or Ctrl+. (Windows/Linux) to toggle between modes
  • Accept Suggestions: Click on mode switch suggestions that Bob offers during conversations

Mode Selection Guidelines

Follow these guidelines to choose the most effective mode for your workflow.

🚀

Starting New Projects

Begin with Plan mode to establish architecture and design before diving into implementation.

Example: "I need to build a REST API for a todo application"
→ Use Plan mode to design endpoints, data models, and architecture
💼

Daily Development

Use Code mode for routine coding tasks—it's cost-efficient and handles most development needs.

Example: "Fix the bug in the login validation"
→ Use Code mode for straightforward bug fixes
📚

Learning & Understanding

Switch to Ask mode when you need explanations without risking file modifications.

Example: "How does this caching mechanism work?"
→ Use Ask mode to safely explore and learn
⚡

Power User Tasks

Use Advanced mode when you need unrestricted access to all tools, especially for MCP integrations.

Example: "Integrate Slack notifications with our CI/CD pipeline"
→ Use Advanced mode for complex integrations
🎯

Complex Features

Use Orchestrator mode for multi-step projects that span different domains and require coordination.

Example: "Build a complete user authentication system with email verification"
→ Use Orchestrator mode to coordinate planning, implementation, and testing
🔄

Mode Switching

Don't hesitate to switch modes mid-conversation—Bob will suggest switches when appropriate.

Example: Start in Ask mode to understand, then switch to Code mode to implement
→ Seamless transitions optimize your workflow

Pro Tips

  • Trust Bob's suggestions: When Bob suggests switching modes, it's usually for good reason
  • Start conservative: Begin with Ask or Plan mode, then move to Code or Advanced as needed
  • Use keyboard shortcuts: ⌘+. or Ctrl+. for quick mode toggling
  • Experiment: Try different modes for the same task to find what works best for your workflow

Troubleshooting

Common issues and solutions when working with custom modes.

Mode Not Loading

Symptoms: Mode appears in list but doesn't activate

Solutions:

  • Check JSON syntax with `bob validate-mode [file]`
  • Verify required fields are present
  • Clear Bob cache: `bob cache --clear`
  • Restart Bob IDE

Tools Not Working

Symptoms: Configured tools don't execute or show errors

Solutions:

  • Verify tool installation: `bob check-tools`
  • Check tool paths in configuration
  • Review permissions for tool execution
  • Test tools manually outside Bob

Performance Issues

Symptoms: Mode responses are slow or timeout

Solutions:

  • Reduce context window size
  • Optimize system prompt length
  • Disable unused integrations
  • Check network connectivity for API calls

Team Sync Issues

Symptoms: Team members have different mode versions

Solutions:

  • Force sync: `bob sync --force`
  • Check Git repository status
  • Verify distribution method is working
  • Review team access permissions
