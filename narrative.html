Analyze the Spec

⏱️ ~30 seconds

Your Goal

Understand what the business is asking for and how it maps to your technical systems.

💬

Ask Bob:

Please analyze the SSO_MFA_Integration.pdf spec and tell me:
1. What systems are involved?
2. What are the current security gaps?
3. What are the SOC 2 compliance requirements?
4. What technology stacks are we working with?

What Bob Returns

📊 Systems Overview

3 repositories identified:

  • auth-service (Node.js/Express) - OAuth token issuance
  • user-service (Java/Spring Boot) - User management, authentication
  • frontend (React) - Customer portal UI

⚠️ Security Vulnerabilities Found

  • Hardcoded JWT secret in tokenController.js
  • CSRF protection disabled in SecurityConfig.java
  • Authentication tokens stored in localStorage (XSS vulnerable)
  • No MFA enforcement (SOC 2 CC6.1 requirement)
  • No session timeout (SOC 2 CC6.6 requirement)

📋 SOC 2 Compliance Requirements

  • CC6.1-2026-01: Implement MFA for all users
  • CC6.6-2026-03: Enforce 15-minute session timeout
  • CC7.2-2026-04: Add authentication event logging
  • CC8.1-2026-05: Remove hardcoded secrets

Plan the Implementation

⏱️ ~45 seconds

Your Goal

Get a concrete implementation plan across all 3 repositories.

💬

Ask Bob:

Create an implementation plan for all 3 systems.

For each system, show:
- Files to create (with purpose)
- Files to modify (with what changes)
- Dependencies to add
- Deployment order and why

Be specific about file paths.

Implementation Plan

SYSTEM 1: auth-service (Node.js)

DEPLOY FIRST

Why first: Other systems depend on these new endpoints

Files to CREATE:
  • src/controllers/mfaController.js → MFA enrollment + verification
  • src/services/totpService.js → TOTP secret generation, QR codes
  • src/middleware/sessionTimeout.js → 15-minute timeout enforcement
  • src/utils/auditLogger.js → Auth event logging
Files to MODIFY:
  • src/controllers/tokenController.js → Add MFA claim to JWT
  • src/routes/auth.js → Add /enroll-mfa, /verify-mfa routes
  • package.json → Add: speakeasy, qrcode libraries

SYSTEM 2: user-service (Java)

DEPLOY SECOND

Why second: Needs auth-service endpoints to exist first

Files to CREATE:
  • config/OAuth2Config.java
  • controller/MFAController.java
  • security/TokenValidator.java
Files to MODIFY:
  • config/SecurityConfig.java → Replace JDBC with OAuth 2.0
  • pom.xml → Add spring-boot-starter-oauth2-client

SYSTEM 3: frontend (React)

DEPLOY LAST

Why last: Backend must support OAuth before frontend uses it

Files to CREATE:
  • src/components/OAuthFlow.jsx
  • src/components/MFAEnrollment.jsx
  • src/components/OAuthCallback.jsx
  • src/services/oauthService.js
Files to MODIFY:
  • src/components/Login.jsx → Remove direct POST
  • src/services/authService.js → Use OAuth flow
