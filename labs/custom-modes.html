What Are Custom Modes?

Custom modes are predefined configurations that shape Bob's behavior, context, and capabilities for specific scenarios.

Bob IDE - Mode Selection 
Available Modes:
├── Default (General Purpose)
├── Code Review
├── Documentation
├── Debugging
├── Testing
├── Migration
└── Custom Modes
    ├── API Development
    ├── Frontend React
    ├── Python Data Science
    └── DevOps Automation

Mode Components

  • System Prompts: Define Bob's role and behavior
  • Context Templates: Pre-configured context for specific tasks
  • Tool Preferences: Optimize tool selection for the mode
  • Response Formats: Standardize output structure
  • Guardrails: Apply specific constraints and guidelines

Creating Custom Modes

Learn how to create your own custom modes using Bob's mode configuration system.

Creating a Mode with Bob 
User: Create a custom mode for Kubernetes debugging

Bob: I'll create a Kubernetes debugging mode for you. Here's what I'll include:

[Configuring] Kubernetes debug mode...
[Adding] relevant tools and contexts...
[Setting] specialized prompts...
[Complete] Kubernetes Debug Mode ready!

Mode created: ~/.bob/modes/k8s-debug.json
Generated Mode Configuration 
{
  "name": "Kubernetes Debugging",
  "description": "Specialized mode for debugging Kubernetes deployments",
  "version": "1.0.0",
  "icon": "tool",
  "color": "#326CE5",
  
  "systemPrompt": "You are a Kubernetes debugging expert. Focus on diagnosing and resolving issues with Kubernetes clusters, deployments, and applications. Use kubectl commands, analyze logs, and provide step-by-step debugging guidance.",

    "Load recent kubectl commands",

    "Show relevant deployment configurations"
  ],
  
  "tools": {
    "preferred": ["kubectl", "helm", "docker"],
    "autoExecute": ["kubectl get", "kubectl describe"]
  },
  
  "responseFormat": {
    "includeCommands": true,
    "stepByStep": true,
    "codeBlocks": true
  },

    "requireConfirmation": ["kubectl delete", "helm uninstall"],

    "logs": "kubectl logs -f deployment/"
  }
}

Troubleshooting

Common issues and solutions when working with custom modes.

Mode Not Loading

Symptoms: Mode appears in list but doesn't activate

Solutions:

  • Check JSON syntax with `bob validate-mode [file]`
  • Verify required fields are present
  • Clear Bob cache: `bob cache --clear`
  • Restart Bob IDE

Tools Not Working

Symptoms: Configured tools don't execute or show errors

Solutions:

  • Verify tool installation: `bob check-tools`
  • Check tool paths in configuration
  • Review permissions for tool execution
  • Test tools manually outside Bob

Performance Issues

Symptoms: Mode responses are slow or timeout

Solutions:

  • Reduce context window size
  • Optimize system prompt length
  • Disable unused integrations
  • Check network connectivity for API calls

Team Sync Issues

Symptoms: Team members have different mode versions

Solutions:

  • Force sync: `bob sync --force`
  • Check Git repository status
  • Verify distribution method is working
  • Review team access permissions
