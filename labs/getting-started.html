Prompt Engineering

Essential Skill

What You'll Learn

Prompt engineering helps you get the most out of Bob by crafting clear, effective instructions. When you write better prompts, you spend less time clarifying your requests and more time getting the results you need.

General Principles

1

Be Clear and Specific

State exactly what you want Bob to do:

Ineffective vs. Effective 
- Ineffective: Fix the code.
+ Effective: Fix the bug in the calculateTotal function that causes it to return incorrect results.
2

Provide Context

Use Context Mentions to refer to specific files, folders, or problems:

Context Example 
@/src/utils.ts Refactor the calculateTotal function to use async/await.
3

Break Down Tasks

Divide complex tasks into smaller, well-defined steps:

Task Breakdown 
Help me implement user authentication:
1. First, analyze the current user model
2. Then, create authentication middleware
3. Finally, add protected routes
4

Give Examples

Provide examples when you have a specific coding style or pattern in mind:

Example with Pattern 
Create a data validator following this pattern:
const validateUser = (data) => {
  const { error, value } = schema.validate(data);
  if (error) return { isValid: false, errors: error.details };
  return { isValid: true, value };
};

Now create a similar validator for product data.
5

Specify Output Format

Indicate if you need the output in a particular format:

Format Specification 
List all API endpoints in this project.
Return the response as a JSON array with this structure:
[
  {
    "method": "GET",
    "path": "/api/users",
    "description": "Get all users"
  }
]

Thinking vs. Doing

Guide Bob through a "think-then-do" process for better results:

Analyze

Ask Bob to analyze the current code, identify problems, or plan the approach:

Analyze Step 
@/src/payment.js Analyze this code for potential bugs and security issues.
Don't fix anything yet, just report what you find.

Plan

Have Bob outline the steps it will take to complete the task:

Plan Step 
Based on the issues found, create a step-by-step plan to fix them.
Prioritize security issues first.

Execute

Instruct Bob to implement the plan, one step at a time:

Execute Step 
Now implement step 1 of the plan: fix the SQL injection vulnerability.

Review

Review the results of each step before proceeding:

Review Step 
Good! Now continue with step 2: add input validation.

Handling Ambiguity

When your request lacks sufficient detail, Bob might make assumptions or ask questions:

Ambiguous Prompt vs. Clear Prompt

Prompt Comparison 
- Ambiguous: Update the user service
+ Clear: Update the user service to include profile picture upload functionality.
+ Use AWS S3 for storage, validate image format, and update the user model.

Providing Feedback

If Bob doesn't produce the desired results:

Reject Actions

Click the "Reject" button when Bob proposes an action you don't want

Provide Explanations

When rejecting, explain why to help Bob learn from mistakes

Reword Your Request

Try rephrasing your initial task or providing more specific instructions

Manually Correct

For small issues, directly modify the code before accepting the changes

Tips and Best Practices

  • Start broad, then narrow: Begin with a general description of what you want to accomplish, then add specific details
  • Use technical terminology: Bob understands programming concepts, so use proper technical terms
  • Mention constraints: State specific requirements or limitations upfront
  • Request explanations: Ask Bob to explain its reasoning or approach when solving complex problems
  • Provide relevant context: Include information about your project's architecture, dependencies, or requirements when necessary

"By following these guidelines, you can write effective prompts that get the most out of Bob's capabilities and help you accomplish your tasks more efficiently."

Bob Generates Context-Aware Code

Generated Code 
// Bob detected your project uses Express.js and async/await
// Bob noticed your error handling pattern and matched it
// Bob included validation based on your existing user model

const { body, validationResult } = require('express-validator');
const User = require('../models/User');
const authUtils = require('../utils/auth');

exports.authenticate = async (req, res) => {
  try {
    // Input validation following your project's pattern
    const errors = validationResult(req);
    if (!errors.isEmpty()) {
      return res.status(400).json({
        success: false,
        errors: errors.array()
      });
    }

    const { email, password } = req.body;
    
    // Authentication logic
    const user = await User.findOne({ email });
    if (!user || !await authUtils.comparePassword(password, user.password)) {
      return res.status(401).json({
        success: false,
        message: 'Invalid credentials'
      });
    }

    // Token generation matching your existing approach
    const token = authUtils.generateToken(user);
    
    res.status(200).json({
      success: true,
      data: {
        user: {
          id: user._id,
          email: user.email,
          name: user.name
        },
        token
      }
    });
  } catch (error) {
    // Error handling consistent with your project
    res.status(500).json({
      success: false,
      message: 'Authentication failed',
      error: process.env.NODE_ENV === 'development' ? error.message : undefined
    });
  }
};

Key points: Bob matched your project's error handling pattern, used existing models, and followed your API response format.

Refined Solution 
// Refactored based on your feedback

const disposableDomains = require('../data/disposableDomains');
const { EmailValidator } = require('email-validator');

const validateEmail = async (email, options = {}) => {
  const {
    checkDisposable = true,
    allowInternational = true,
    detailedErrors = true
  } = options;

  // Basic format validation
  if (!email || typeof email !== 'string') {
    return {
      isValid: false,
      error: detailedErrors ? 'Email is required and must be a string' : 'Invalid email'
    };
  }

  // International domain support
  if (allowInternational && !EmailValidator.validate(email)) {
    return {
      isValid: false,
      error: detailedErrors ? 'Email format is invalid' : 'Invalid email'
    };
  }

  // Check for disposable domains
  if (checkDisposable) {
    const domain = email.split('@')[1]?.toLowerCase();
    if (disposableDomains.includes(domain)) {
      return {
        isValid: false,
        error: detailedErrors ? 'Disposable email addresses are not allowed' : 'Invalid email'
      };
    }
  }

  // Additional business rules can be added here
  
  return {
    isValid: true,
    normalizedEmail: email.toLowerCase().trim()
  };
};

module.exports = { validateEmail };
