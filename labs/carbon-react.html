    Provide Feedback

    If Bob's implementation doesn't match your expectations, provide specific corrections and examples.

    Good Feedback: "The spacing between form fields should be 32px instead of 24px. Also, use the 'lg' size variant for the TextInput components."

    Better Feedback: "Update the form to use spacing-07 (32px) between fields and set size='lg' on all TextInput components. Reference the Carbon spacing scale documentation."
    📚

    Use Carbon Terminology

    Refer to components by their Carbon names for better results and clearer communication.

    Carbon Component Names:
    • ClickableTile (not "clickable card")
    • TextInput (not "text field")
    • DataTable (not "table")
    • Modal (not "dialog" or "popup")
    • Accordion (not "collapsible section")
    📱

    Test Responsiveness

    Always verify that components work correctly across different screen sizes and breakpoints.

    Carbon Breakpoints:
    • sm: 320px - 671px (mobile)
    • md: 672px - 1055px (tablet)
    • lg: 1056px - 1311px (desktop)
    • xlg: 1312px - 1583px (large desktop)
    • max: 1584px+ (extra large)
    ♿

    Accessibility First

    Carbon components are accessible by default, but ensure you're using them correctly.

    Accessibility Checklist:
    • Provide meaningful labels for all inputs
    • Use helper text for additional context
    • Ensure proper heading hierarchy (h1, h2, h3)
    • Add alt text for images and icons
    • Test keyboard navigation
    • Verify screen reader compatibility
    🎨

    Follow Design Tokens

    Use Carbon's design tokens for spacing, colors, and typography to maintain consistency.

    Common Design Tokens:
    • Spacing: spacing-03 (8px), spacing-05 (16px), spacing-07 (32px)
    • Colors: Use semantic color tokens (e.g., text-primary, background-hover)
    • Typography: Use type tokens (e.g., heading-01, body-short-01)
