Getting Started

Foundation

Setting Up Your Environment

Before using Carbon React mode, ensure your project has the necessary dependencies.

Install Dependencies 
# Install Carbon React components
npm install @carbon/react @carbon/themes @carbon/icons-react

# Or with yarn
yarn add @carbon/react @carbon/themes @carbon/icons-react

Project Structure

Ensure your project is set up to support Carbon components:

  • React 16.8+ (for hooks support)
  • Sass support (for Carbon styles)
  • Modern bundler (webpack, vite, etc.)

Core Skill

Describing UI Components

Learn how to effectively describe UI components for Carbon React implementation.

Be Specific

  • Specify exact Carbon component names
  • Include props and variants
  • Define spacing and layout
  • Mention interaction states

Provide Context

  • Describe the user flow
  • Explain data requirements
  • Specify responsive behavior
  • Include accessibility needs
Example Specification 
Create a login form using Carbon React components:

1. Use a Form component with spacing-07 between fields
2. Add a TextInput for email with type="email" and labelText="Email"
3. Add a PasswordInput with labelText="Password" and helperText="Minimum 8 characters"
4. Include a Checkbox with labelText="Remember me"
5. Add a Button component with kind="primary" and text="Sign in"
6. Use Grid system with 4 columns offset on large screens
7. Make it responsive: full width on mobile, 6 columns on tablet

Implementation Process

Workflow

Step-by-Step Implementation

Follow this structured approach for best results with Carbon React mode.

Step 1: Analyze the Design

Break down your UI into Carbon components and identify the structure.

Component Analysis 
Header: Header (fixed position)
Navigation: HeaderNavigation
Content: Grid + Row + Column
Form: Form with TextInput, Button, etc.
Footer: Footer (simple)

Step 2: Implement Section by Section

Build your UI incrementally, starting with the layout structure.

Implementation Order 
1. Set up the basic layout structure
2. Add major components (Header, Footer)
3. Implement the main content area
4. Add interactive elements
5. Apply responsive design
6. Refine spacing and styling
4

Best Practices

Guidelines

Start with Layout

Always begin with Grid, Row, and Column components for proper structure.

Example: "Create a 16-column grid with a sidebar that spans 4 columns on large screens."

Use Carbon Props

Leverage built-in props for styling instead of custom CSS.

Good: Use `kind="secondary"` on Button
Bad: Custom CSS to change button color

Design Tokens

Use Carbon's design tokens for consistent spacing and sizing.

Common Tokens: spacing-01 (4px), spacing-03 (8px), spacing-05 (16px), spacing-07 (32px)

Theme Support

Implement with white and g10/g90/g100 theme variants in mind.

Tip: Use layer tokens instead of hardcoded colors for theme compatibility
