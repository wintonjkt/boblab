What Are Custom Rules?

5 minutes

Custom rules extend Bob's default behavior by defining preferences, constraints, and guidelines that shape how Bob interacts with your code and responds to your requests.

Understanding Custom Rules

Example 
{
  "name": "Java Style Guide",
  "scope": "project",
  "rules": [
    {
      "description": "Use proper naming conventions",
      "pattern": "^[a-z][a-zA-Z0-9]*$",
      "enforcement": "error"
    },
    {
      "description": "Maximum method length",
      "pattern": "^[a-zA-Z_][a-zA-Z0-9_]{1,50}$",
      "enforcement": "warning"
    }
  ]
}

💡 Key Concepts

Custom rules use JSON configuration with pattern matching and enforcement levels (error, warning, info).

Common Use Cases

  • Coding Standards: Enforce consistent formatting, naming, and structure
  • Security Rules: Prevent dangerous patterns and enforce secure coding
  • Project Guidelines: Ensure team-wide consistency in approaches

Adding Custom Rules

10 minutes

Creating Your First Rule Set

1

Open Bob Settings

Navigate to Bob Settings → Custom Rules to create your first rule set.

menu 
Settings → Bob Rules → Create Rule Set
2

Define Rule Scope

Choose whether rules apply globally, to specific projects, or only in certain modes.

json 
{
  "scope": "workspace",
  "mode": "java",
  "priority": 1
}

⚠️ Pro Tip

Start with a small rule set and gradually expand as you understand Bob's behavior with your specific workflows.

3

Rule Scopes

5 minutes

Custom rules can be applied at different levels to control when and where they take effect.

Global

Apply to all files and projects regardless of location

Workspace

Apply to all files in the current workspace

Project

Apply only to files within specific project folders

Mode-Specific

Apply only when Bob is in specific modes

Real-World Examples

10 minutes

Practical examples of custom rules used in enterprise environments:

Security-Focused Rules

json 
{
  "name": "Security Hardening",
  "rules": [
    {
      "description": "Prevent SQL injection vulnerabilities",
      "pattern": "\\b(SELECT|INSERT|UPDATE|DELETE)\\b",
      "enforcement": "error",
      "exceptions": ["prepared-statements", "parameterized-queries"]
    },
    {
      "description": "Require secure authentication patterns",
      "pattern": "(password|token|secret)",
      "enforcement": "error",
      "context": "auth"
    }
  ]
}

✅ Result

These rules automatically detect and prevent common security vulnerabilities while allowing exceptions for legitimate use cases.

Performance Optimization Rules

json 
{
  "name": "Performance Guidelines",
  "rules": [
    {
      "description": "Optimize for common patterns",
      "patterns": [
        {
          "match": "for.*\\{.*\\}",
          "suggest": "enhanced-for-loop"
        },
        {
          "match": "String\\+.*String",
          "suggest": "StringBuilder"
        }
      ],
      "enforcement": "warning"
    }
  ]
}

🚀 Impact

Automatically suggests performance optimizations while educating developers on best practices.
