Scene 5

Security Scanning

Real-time

Learning Objective

Experience Bob's inline security scanning that catches vulnerabilities during code generation, not days later in reviews.

🚨 Security Detection Example

What Bob Generated First (with issue)
// Bob noticed this pattern:
const jwt = require('jsonwebtoken');
const token = jwt.sign(payload, 'secret-key');  // ← Hardcoded!
What Bob Fixed Automatically
// Bob self-corrected:
const jwt = require('jsonwebtoken');
const token = jwt.sign(payload, process.env.JWT_SECRET);  //  Environment var

Security Issues Bob Caught

Hardcoded JWT Secret

Fixed by using environment variable

CSRF Protection Disabled

Fixed by enabling with proper configuration

Tokens in localStorage

Fixed by using httpOnly secure cookies

Missing Session Timeout

Fixed by adding 15-minute enforcement

Verification Checkpoint

Bob detected and fixed:

  • 4 security vulnerabilities
  • During generation (not in review)
  • With explanations for each fix

Seller Talking Point

Security Inline:

  • Traditional: Security review happens days later (find more issues, more rework)
  • With Bob: Security scanning happens during generation (zero debt accumulation)

"Shift-left security - vulnerabilities are caught before code is written, eliminating the security-review-rework cycle."
