What is Bob-a-thon?

Bob-a-thon is a hands-on workshop that demonstrates IBM Bob's enterprise capabilities through a realistic scenario: implementing SSO + MFA across 3 legacy systems to pass a SOC 2 audit. Experience firsthand how Bob transforms 7-8 weeks of traditional development into 10 minutes of AI-powered orchestration.

⏱️ Time Required

45-60 minutes (self-paced)

🎯 Format

Hands-on, step-by-step guided experience

📊 Level

Beginner to Advanced (all welcome)

Learning Objectives

  • Understand how Bob reads and interprets business requirements documents
  • Experience multi-repository orchestration across different tech stacks
  • See compliance automation in action (SOC 2, PCI-DSS)
  • Witness security vulnerabilities caught and fixed inline
  • Learn deployment planning with zero-downtime strategies
  • Explore Bobalytics for cost transparency and productivity tracking

Workshop Journey

Follow this progressive journey through Bob's capabilities:

Stage 0: Just Got the Project

Start with a realistic scenario: You've been assigned to implement SSO + MFA across legacy systems to pass a SOC 2 audit. The deadline is tight, and you have a 20-page requirements document.

  • Review the business requirements document
  • Understand the compliance requirements
  • Identify the affected systems

Stage 0.5: The Existing Codebase

Explore the three legacy repositories you'll be working with:

  • auth-service (Node.js) - Authentication microservice
  • user-service (Java) - User management system
  • frontend (React) - Customer-facing application

Stages 1-8: Implementation Journey

Work through each capability demonstration:

  1. Business Requirements Analysis - Upload the PDF and let Bob extract technical requirements
  2. Compliance Mapping - Map SOC 2 findings to specific code changes
  3. Multi-Repo Planning - Generate implementation plan across all 3 repositories
  4. Code Generation - Watch Bob create 17+ files with proper integration
  5. Security Inline - See vulnerabilities caught and fixed in real-time
  6. Test Generation - Comprehensive test suites with 94% coverage
  7. Deployment Planning - Zero-downtime deployment runbook
  8. Compliance Evidence - Automated audit documentation

Stage 9: Bobalytics Dashboard

Review the complete productivity and cost analysis:

  • Task completion time: 10 minutes vs 7-8 weeks traditional
  • Total cost: $8.47 vs $192,000 traditional
  • Files generated: 17+ across 3 repositories
  • Test coverage: 94% production-ready
  • Security issues: 3 detected and fixed inline

Learning Paths

For Individual Developers

Focus on hands-on technical capabilities:

  • Work through all 9 stages at your own pace
  • Experiment with different prompts and approaches
  • Explore the generated code and understand Bob's decisions
  • Try modifying requirements and see how Bob adapts
  • Review Bobalytics to understand cost and productivity metrics

For Team Leaders/Facilitators

Run the workshop as a team exercise:

  • Allocate 60-90 minutes for the full workshop
  • Have participants work in pairs or small groups
  • Pause at each stage to discuss observations and insights
  • Compare results across teams (different approaches, same outcome)
  • Conclude with a discussion on how to integrate Bob into your workflow

For Sales/Leadership

Focus on business value and ROI:

  • Review the README and Stage 9 (Bobalytics) first
  • Understand the time and cost comparisons
  • Note the compliance automation capabilities
  • Use the metrics for customer conversations
  • Reference the IBM internal success stories
